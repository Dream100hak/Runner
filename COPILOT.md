# GitHub Copilot 지시 파일 (COPILOT.md)

목적
- 프로젝트 전반에 대해 GitHub Copilot가 일관된 스타일, 안전 규칙, 및 컨텍스트를 따르도록 지시합니다.
- Unity + C#(C# 9.0, .NET Framework 4.7.1) 환경에 최적화된 출력 생성.

적용 범위
- 기본: 리포지토리 루트에 위치한 이 파일을 우선 참조합니다.
- 코드 베이스: 특히 `Assets/` 폴더와 `Assets/Scripts/` 내 파일들을 우선 고려합니다.

행동 원칙
- 간결하고, 명확하며, 실용적인 코드를 생성합니다.
- Unity API 사용 시 성능 및 메모리 비용을 고려합니다(예: Update()에서 불필요한 할당 회피).
- 변경 제안은 최소한의 영향으로 안전하게 적용되도록 권장합니다.

코딩 스타일 (우선순위)
- 언어: C# 9.0, 타깃: .NET Framework 4.7.1
- 네이밍: PascalCase(클래스/메서드/프로퍼티), camelCase(로컬 변수, 매개변수), UPPER_SNAKE_CASE(상수)
- 전역 변수는 _를 붙여주도록 해_
- 접근성: 가능한 한 최소한의 접근 제한자 사용(필요 시 public)
- 주석: 공개 API 및 복잡한 로직에 XML 문서 주석 사용
- Unity 규약:
  - MonoBehaviour 이벤트 메서드는 필요한 경우에만 오버라이드
  - 성능 민감 루프에서는 반복 할당을 피함
  - SerializeField + private 조합 권장(인스펙터 노출 목적)

안전/보안
- 사용자 입력/네트워크 데이터는 항상 유효성 검사
- 민감 정보(키, 비밀번호)를 하드코딩하지 않음
- 외부 라이브러리 사용 시 라이선스와 보안 취약점 표기

출력 규칙
- 새 파일 생성 시 파일 헤더에 간단한 설명과 작성자(자동완성 출처)를 포함
- 변경 제안(PR 포맷 예시: 제목 한 줄, 요약, 변경 파일 목록)
- 테스트 코드를 함께 제안 가능(가능할 때 NUnit/Unity Test Runner 사용)

예제 프롬프트 템플릿
- "이 파일을 Unity 6000.2. 환경에 맞게 최적화해줘. 주요 변경사항만 PR 코멘트로 설명해줘."
- "C# 9.0 규약을 적용해서 리팩터링 해줘. 비동기/동기 경계와 안전성에 주의해줘."

금지 항목
- 라이브러리/코드의 전체 복제(저작권 침해 가능)
- 민감 데이터 포함
- 프로젝트 컨텍스트와 무관한 대규모 변경(사용자 지시 없는 경우)

커밋 메시지 스타일 (권장)
- 형식: `<영역>: <간단한 설명>`  
- 예: `Player: Avoid per-frame allocation in Update`

디버깅 및 테스트
- 변경 제안 시 가능한 간단한 재현 단계 제공
- Unity 편집기에서 확인할 테스트 절차(예: 플레이 모드 진입, 특정 입력 시 동작 확인) 제시

Visual Studio 설정(참고)
- Copilot 확장 확인: __Extensions > GitHub Copilot__
- Git 관련 기본 설정: __Tools > Options > Source Control__

메타
- 작성자: GitHub Copilot 지침 (자동 생성)
- 업데이트: 필요 시 팀 컨센서스에 따라 수정
